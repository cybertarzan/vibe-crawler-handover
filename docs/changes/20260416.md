## 수정 파일

### Backend
- **의존성 어댑터 전면 리팩토링 (`shared/config/base.py` 및 도메인 레벨 전반)**
  - `workflow/nodes/`(`llm`, `rag`, `vlm`, `mcp`, `python_scripter`), `admin/`, `logger` 내 파편화되어 존재하던 모든 `os.getenv` 코드를 전면 제거.
  - 전 도메인이 의존성 주입(DI)을 통해 `Settings` 의 중앙 집중식 설정을 사용하도록 아키텍처 수준의 대규모 리팩토링 진행.
- **크롤러 코어 내재화 신설 (`resources/crawler_core/` 전역)**
  - `script_crawler.py`, `screenshot_manager.py` 등 핵심 크롤러 구동기 파일 10여 개 신규 추가. 오프라인 복사(Scp 등) 배포 의존성을 걷어내고 백엔드를 '단일 진실 공급원'으로 승격.
- **스케줄러 모듈 및 어댑터 개선 (`workspace/project/job/crawling/` 및 `scheduler/`)**
  - `docker_crawler_adapter.py`, `local_crawler_adapter.py`: 실행 직전 내재화된 `crawler_core`를 강제 핫픽스로 주입하도록 어댑터 동작 매커니즘 고도화.
  - `crawling_executor_adapter.py`: API 통신 포트를 고정(8080)하지 않고 명시된 `API_BASE_URL` 로 유연하게 호출하도록 변경 (로컬/운영 네트워크 단절 에러 해결)
  - `crawler_path_service.py`, `file_system_adapter.py`: 파일 조작 및 저장소 시스템 포트 연동 고도화.
- **인프라 의존성 업그레이드 (`Dockerfile` 및 `prod.py`)**
  - 데비안 의존성 문제로 뻗는 Playwright 설치 시스템(`playwright install-deps`)을 완벽 호환시키기 위해 베이스 이미지를 `python:3.11-slim-bookworm` 으로 교체.
